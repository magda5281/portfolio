## Evolution

1. **Prop Drilling**  
   All state and update handlers lived in `App.jsx` and were passed down through
   multiple layers. Functional components with props & `children` composition
   were used originally
2. **Context API + Custom Hook**  
   Introduced an `ItemsContextProvider` and `useItemsContext` to avoid
   prop-drilling, with manual localStorage sync.
3. **Zustand Store**  
   Replaced Context entirely with a lightweight Zustand store (`itemsStore.js`)
   using its `persist` middleware to handle localStorage automatically.
   > The old `contexts/` folder and `lib/hooks.js` remain in the repo for
   > reference only.
